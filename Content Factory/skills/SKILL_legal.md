# SKILL: Legal — Юридические правила атрибуции

## Назначение
Этот skill описывает, как система работает с чужим контентом законно и без юридических конфликтов.

---

## Основной принцип

Система **не скачивает и не перезаливает** чужие видео.  
Система публикует **ссылку на оригинал** с именем автора — это репост/рекомендация, а не присвоение.

---

## Что разрешено (без разрешения автора)

| Действие                              | Статус      |
|---------------------------------------|-------------|
| Поделиться ссылкой на видео           | ✅ Разрешено |
| Написать обзор чужого видео           | ✅ Разрешено |
| Использовать скриншот/превью          | ✅ Разрешено (fair use) |
| Процитировать короткий фрагмент текста| ✅ Разрешено с атрибуцией |
| Embed YouTube-видео на сайт           | ✅ Разрешено |

---

## Что запрещено

| Действие                                    | Статус       |
|---------------------------------------------|--------------|
| Скачать и перезалить видео на свой канал    | ❌ Запрещено  |
| Обрезать видео и выдать за своё             | ❌ Запрещено  |
| Удалить водяной знак автора                 | ❌ Запрещено  |
| Публиковать без указания автора             | ❌ Запрещено  |
| Монетизировать чужой контент                | ❌ Запрещено  |

## Обязательный формат атрибуции

Каждый пост, основанный на чужом материале, **обязан** содержать:

```
📹 Видео: {Имя автора / Название канала}
🔗 Оригинал: {прямая ссылка на видео}
```

**Примеры:**

Instagram:
```
📹 Видео: Cruise With Ben And David
▶️ Смотреть оригинал — ссылка в bio
```

YouTube Community:
```
📹 Видео: Cruise With Ben And David
🔗 https://youtube.com/watch?v=abc123
```

## Лицензии YouTube-видео

YouTube-видео по умолчанию защищены авторским правом. Однако репост **ссылки** (не самого файла) является законным во всех юрисдикциях.

### Как определить лицензию видео через API
```
GET https://www.googleapis.com/youtube/v3/videos
  ?part=status
  &id={videoId}
  &key={YOUTUBE_API_KEY}

Ответ: status.license = "youtube" | "creativeCommon"
```

- `youtube` → стандартная лицензия, только репост ссылки
- `creativeCommon` → Creative Commons, можно использовать шире

---

## Приоритизация контента

При прочих равных условиях выбирать контент:
1. Creative Commons (`creativeCommon`) — высший приоритет
2. Видео с YouTube с явным разрешением в описании
3. Стандартная лицензия — только репост ссылки

---

## Что делать при DMCA-жалобе

1. Немедленно удалить пост (вручную Сырымом или Сайрой)
2. Добавить `source_url` в таблицу `content_items` с полем `status = 'blocked'`
3. Система больше никогда не будет использовать этот URL

```sql
UPDATE content_items
SET status = 'blocked'
WHERE source_url = '{url}';
```

---

## Проверка перед публикацией (чеклист)

- [ ] Присутствует `author_name` (не пустой)
- [ ] Присутствует `author_url` (не пустой)  
- [ ] Присутствует `source_url` (оригинальная ссылка)
- [ ] Текст поста содержит блок атрибуции
- [ ] Видеофайл НЕ скачивается и НЕ перезаливается

Если хотя бы один пункт не выполнен — **публикация блокируется**.
