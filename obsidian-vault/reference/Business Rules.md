# Бизнес-правила (нарушать нельзя)

## Данные
1. **Contact ≠ Lead** — лиды только через форму лендинга, контакты = телефонная книга партнёра
2. **Lead → Contact конвертация запрещена (постоянно).** Лид навсегда остаётся в `leads`, статус меняется (qualified/converted/...), row не уходит. WF7 (Lead-to-Contact) удалён 2026-05-06. **Contact → Lead** = future Phase C feature: только через системный path, когда партнёр запускает презентацию для контакта из книги (не вручную dashboard-кнопкой)
3. Max 25 таблиц в Supabase (сейчас 19, см. CLAUDE.md / `list_tables`)
4. Старый проект (66 таблиц, `ap-southeast-2`) — **мёртв**, не использовать

## Копирайтинг и compliance
5. **Никаких гарантий дохода** — требование InCruises + Meta/WABA
6. Scarcity tactics запрещены ("мест ограничено")
7. Прямое название MLM компании убрано из рекламного копирайтинга
8. Вся коммуникация должна быть Meta-safe
9. **AI-агент работает в полуавтомате** — LLM не отправляет сообщения лидам напрямую. Только генерирует черновик → партнёр редактирует/одобряет → отправка. Причина: WABA банит за нарушения Meta policy, LLM может непреднамеренно пообещать доход
10. **Gate активации AI: обязательное обучение** — AI-агент НЕ активируется для партнёра, пока не закончил обязательные модули («Старт партнёра», «Compliance: что нельзя обещать», «Дашборд SYSTEM V2», «AI-агент: работа с черновиками»). Защита от compliance-нарушений. Принято 2026-04-18
