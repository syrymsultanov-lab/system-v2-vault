## Решения

- **Двойная архитектура для AI диалога**: реактивный inline pipeline в Inbound (latency 3-5с), Conversation Loop остаётся cron для proactive (приветствия/followup, latency некритична). **Why:** TG webhook ждёт 60с, OpenAI ~2-3с, всё успевает inline. Cron-polling неприемлемо для реактива.
- **Outbound dispatch**: WF9 переписан для proactive дispatch (cron 1м). Reactive путь шлёт TG напрямую в Inbound, минуя `outbound_messages` queue. **Why:** двойной запас, latency должна быть минимальная.
- **RAG досрочно** (изначально Phase C5, 2-3 сессии): сделано в одной сессии по explicit запросу. **Why:** ответы AI «кастрированные» без знаний → RAG обязательно для quality.
- **Корпус KB**: помимо InCruises-материалов (FAQ/Comp Plan/PDFs) добавлены 5 MLM-классиков (Kalench, Rendy, Don Failla, Big Al, Jim Rohn). **Why:** Сырым хочет, чтобы AI мог говорить с лидом про возможность в принципе, не только про InCruises. Корпус 1.6K чанков из книг.
- **Anti-question fatigue**: q_and_a/presentation/objection без обязательного вопроса в конце. **Why:** Сырым: «слишком частые вопросы». Поведение AI было «каждое сообщение = ответ + новый вопрос».
- **Chunk size 1000 chars** (было 2000). **Why:** более точный retrieval, меньше шума в одном чанке.
- **ivfflat lists=20**: Free Plan лимит памяти на построение индекса. Pro+ позволит `lists=50` (рекомендуемое sqrt(2289)≈48).
- **NO ROTATION**: локальный `OPENAI_API_KEY` в `.env` стал 401 (stale). VPS-ключ валиден. Per `feedback_no_preventive_secret_rotation` — не ротируем превентивно. Локально перепишем при необходимости позже.
- **Без yes/no prompts**: добавлен memory `feedback_no_yes_no_prompts` после явной просьбы Сырыма не уточнять каждое действие.
