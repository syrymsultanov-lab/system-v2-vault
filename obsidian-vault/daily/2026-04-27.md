### Решение: убрать n8n WF1 из критического пути формы
Проанализировали с Сырымом: WF1 — это просто прокси с нормализацией полей, никакой добавленной ценности не даёт. **AI-агент работает через триггер БД** (`leads INSERT → ai_jobs queued → WF3 schedule`), а не через webhook. Значит лид может попадать в Supabase напрямую, и AI-цепочка отработает так же.

Минусы WF1 которые сняли:
- Single point of failure (n8n VPS упал → форма мёртвая)
- CORS-настройка n8n.sairateam.com → sairateam.com (так и не проверена)
- Лишний хоп → задержка submit
- Два места для багов (баг `full_name` vs `name`+`last_name` уже ловили 24-го)

Плюсы прямой записи:
- Anon key уже publishable, RLS защищает чтение
- Меньше движущихся частей
- Одна точка деплоя (FTP), не нужно следить за двумя
