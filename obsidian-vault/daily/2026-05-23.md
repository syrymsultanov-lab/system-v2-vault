### 6. 🔴 КРИТИЧЕСКИЙ BUG — AI 24+ часа молчал из-за моего вчерашнего patch

Сырым: «TG бот замолчал, не отвечает». DB-проверка: outbound пустой 24 часа.

**Root cause #1 — bug в моём `patch_inbound_debounce.py` (вчера):**
- `44_If_Latest` condition был `leftValue: $json.body`
- Supabase RPC `should_process_inbound_now` вернул скалярный `true` напрямую в `$json` (без `.body`)
- IF читал undefined → всегда FALSE branch (workflow ends)
- **Каждый inbound** дропался → 0 outbound

**Root cause #2 — bug в самом patch скрипте:**
- `if all(name in existing) → only re-applying connections` НЕ обновлял parameters существующих нод
- Fix: upsert logic — replace parameters если name match
- Re-applied → IF condition теперь `$json` напрямую

**Effect:** AI снова отвечает существующим контактам (Сайра/Сырым smoke).

### 8. 🟢 Fix — TG bridge (migration + WF patch)

**Migration `tg_lead_to_contact_bridge`:**

1. **Trigger `create_contact_from_lead`** (AFTER INSERT на leads):
   - TG-only, с handle → INSERT contact с `ai_consent=true` (consent с form checkbox), source='landing_form'
   - Idempotent (skip если контакт по handle exists)

2. **RPC `link_tg_contact_by_handle(p_chat_id, p_handle, p_name, p_fallback_partner_id)`** — atomic 3-step upsert:
   - Step 1: find by `tg_chat_id` → return existing
   - Step 2: UPDATE contact по handle SET tg_chat_id = NEW (form lead first bot write)
   - Step 3: INSERT fresh contact с `ai_consent=true` reactive (random TG user без формы)

3. **Backfill** 3 текущих кандидатов → `contacts`

**WF patch `patch_inbound_link_or_create_contact.py`:**
- `20_Find_Contact` GET → POST `/rpc/link_tg_contact_by_handle`
- Body: `{p_chat_id, p_handle: from_handle, p_name: from_username}`
- RPC сам делает find/link/create

**Эффект:**
- Любой TG user → contact автосоздаётся + AI отвечает
- 160 партнёров могут писать боту БЕЗ формы (open-access mode)
- Лендинг-кандидат + первое сообщение боту → handle linked + AI отвечает
- Compliance: ai_consent=true заслужен либо form-checkbox, либо reactive (lead первый написал)
